# MIT License Cheatsheet

The **MIT License** is the most popular, short, and permissive open-source license in existence. It allows developers to do almost anything they want with the code, provided they include the original copyright notice.

## Quick Summary

| Property | Status |
| :--- | :---: |
| **Commercial Use** | ✅ Allowed |
| **Modification** | ✅ Allowed |
| **Distribution** | ✅ Allowed |
| **Sublicensing** | ✅ Allowed |
| **Private Use** | ✅ Allowed |
| **Must Disclose Source** | ❌ No |
| **Must Share Modifications** | ❌ No |
| **Patent Grant** | ❌ None explicit |

---

## What You CAN Do

*   **Use commercially**: Use the code in proprietary, closed-source software sold to customers.
*   **Modify**: Change, refactor, or build upon the library's code.
*   **Distribute**: Package the code with your software and distribute it to others.
*   **Sublicense**: Change the license of your project (you can distribute your project under a proprietary license even if it contains MIT code).

## What You MUST Do

*   **Include License & Copyright Notice**: You must include the original copyright line (e.g., `Copyright (c) 2026 Author Name`) and the license text in all copies or substantial portions of the software.

## What You CANNOT Do

*   **Hold Liable**: The software is provided "as is", without warranty of any kind. You cannot sue the author if the library breaks your application or causes damage.

---

## When to Choose MIT

*   You want your project to be as widely used as possible.
*   You don't care if others modify your code and keep their modifications secret.
*   You want zero friction for commercial adoption.
