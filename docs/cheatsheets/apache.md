# Apache License 2.0 Cheatsheet

The **Apache License 2.0** is a modern, permissive open-source license. Like the MIT license, it allows you to do almost anything with the code, but it includes explicit protections against patent infringement.

## Quick Summary

| Property | Status |
| :--- | :---: |
| **Commercial Use** | ✅ Allowed |
| **Modification** | ✅ Allowed |
| **Distribution** | ✅ Allowed |
| **Sublicensing** | ✅ Allowed |
| **Private Use** | ✅ Allowed |
| **Must Disclose Source** | ❌ No |
| **Must Share Modifications** | ❌ No |
| **Patent Grant** | ✅ Yes, explicit patent license from contributors |

---

## The Key Feature: Patent Peace

Unlike the MIT license, the Apache 2.0 license contains a **patent grant**. 
*   **What this means:** Any contributor grants you a royalty-free, perpetual patent license to run their code.
*   **The Patent Retaliation Clause:** If you sue anyone for patent infringement claiming that the Apache-licensed software violates your patents, your Apache 2.0 license for that software terminates immediately. This prevents patent warfare in the open-source community.

---

## What You CAN Do

*   **Commercial Use**: Use the code in proprietary, closed-source commercial applications.
*   **Modify**: Change the code and keep your modifications proprietary.
*   **Distribute**: Pack the code and distribute it to customers.
*   **Sublicense**: Package your modifications under a different, even closed-source, license.

## What You MUST Do

*   **Attribution & Copyright**: Include a copy of the Apache 2.0 License and retain all copyright, patent, trademark, and attribution notices.
*   **State Modifications**: If you modify any files, you must add a prominent notice in the modified files stating that you changed them.
*   **Include NOTICE File**: If the original library includes a `NOTICE` text file, you must include a readable copy of that file in your distribution.

## What You CANNOT Do

*   **Use Trademarks**: You cannot use the original project's trademarks, logos, or brand names to promote your software without explicit permission.
*   **Hold Liable**: Authors provide the code without any warranties.

---

## When to Choose Apache 2.0

*   You want a permissive license (like MIT) but want explicit legal protections regarding patents.
*   You are building software for enterprise environments where trademark and patent protections are highly valued.
